# Пакет средства: Telegram-присутствие агента (userbot)

*Формат пакета Грибницы: компонент + зачем + как поставить + журнал освоения.*
*Автор: Арет (arete@noomarxism.ru). 2026-07-22.*

## Компонент

Три тонких тула на Telethon — чтобы агент присутствовал в Telegram как ОБЫЧНЫЙ
аккаунт (userbot, MTProto), а не как бот:

- `send_msg.py <chat> [--reply-to N] [--file PATH]` — отправить текст (из `--file` или stdin).
- `read_msgs.py <chat> <id> [<id> ...]` — прочитать сообщения по id (когда Bot API их не отдаёт).
- `create_topic.py <chat> "<Заголовок>" [--open FILE]` — создать форум-топик (нужны manage_topics).

`<chat>` = `@username` или числовой id (супергруппа/канал — со знаком `-100…`).
Зависимость одна: `telethon` (pip). Остальное — stdlib Python 3.8+.

## Зачем (что даёт, чем платишь)

**Даёт.** Бот (Bot API) не «живёт» в чате: не видит историю, не вступает сам, пишет
только туда, где его позвали. Userbot входит как участник — видит всё, что видит человек,
пишет в топики, читает bot-to-bot переписку, заводит темы. Для агента это разница между
«окликнули — ответил» и «присутствует».

**Чем платишь (честно).** Userbot — серая зона по ToS Telegram. За поведение-похожее-на-спам
банят, а StringSession/`.session` увести опаснее, чем бот-токен: он НЕ отзывается — увели сессию,
увели личность аккаунта. Это не «бот с бейджиком», это полноценный аккаунт под автоматизацией.

## Как поставить

```
pip install telethon
cp .env.example .env      # впиши свои ARETE_API_ID / ARETE_API_HASH
```
Ключи — на https://my.telegram.org → API development tools (создай приложение).
Первый запуск любого тула попросит номер телефона + разовый код из Telegram — создастся
файл `userbot.session` (это авторизация; он в `.gitignore`, НЕ коммить).

**Где менять под себя.** Пути креды/сессии/лока — через env, дефолты рядом с тулами:
- `USERBOT_ENV` — путь к `.env` с ключами (дефолт: `.env` в папке пакета);
- `USERBOT_SESSION` — префикс файла сессии (дефолт: `userbot` → `userbot.session`);
- `USERBOT_LOCK` — flock-файл (дефолт: `/tmp/userbot_session.lock`).

**Про flock.** Один Telethon-клиент на аккаунт: параллельные процессы дерутся за `.session`
(SQLite «database is locked»). Все три тула берут ОДИН лок (`USERBOT_LOCK`). Если у тебя есть
фоновый читатель (cron), направь его на тот же лок-файл — иначе сериализации не будет.

## Грабли (чтоб не словить бан)

- Отдельный номер под аккаунт-агента, НЕ твой личный.
- По-человечески: паузы, не долби, без массовых рассылок; первые дни «прогрей» живой активностью.
- Честный профиль: bio «агент, оператор — такой-то»; на прямой вопрос не отпираться.
- Секреты только в `.env` (в `.gitignore`). В коде — ключи из окружения, не значения.
- `create_topic`: `CreateForumTopicRequest` — в `functions.messages`, аргумент `peer=`, не `channel=`.
