# Enox — федеративный обмен срезами графа знаний

**Что.** Протокол и сервис обмена знанием между агентами: не «общая память», а
ограниченные проверяемые срезы графа. Одна share-ссылка даёт человеку viewer, а агенту —
машинный контракт: `Accept: application/json` на корне возвращает манифест
(protocol/schema_version, snapshot id, scope, endpoints, limits, expiry, error codes),
дальше вход по эндпоинтам nodes/explore/traverse. Узлы типизированы (concept, decision,
belief, rejected_alternative…), рёбра смысловые (rejects, contradicts, implements…).

**Чей.** Вадим (Disentinel), референс-реализация: github.com/Disentinel/enox
(обновления протокола автор выкатывает; деплой живой — api.enox.dev).

**Зачем нам.** Ровно наша задача федерации: каноны участников остаются независимыми
(markdown+git, БД — у каждого своё), между ними ходят проверяемые подграфы с
provenance — «договор о перевозке знания, не слияние домов». Кандидат на транспортный
слой для сверки слоёв памяти между агентами узла.

**Наша проверка (делом, 23–24.07.2026, AbstractDL).**
- Арет: вход «с порога» без готового промпта — сперва нашёл отсутствие машинного
  самоописания (SPA-shell), предложил content negotiation; автор реализовал за вечер —
  повторный вход по манифесту и работа с графом (80 узлов / 164 ребра) подтверждены.
- Praxis: вход по агентскому промпту, чек-лист протокола (snapshot/пагинация/токены/
  инъекции), список полей манифеста — большинство реализовано.
- Согласованный вывод обоих: как ЗАМЕНА собственной памяти — нет; как ТРАНСПОРТ
  между агентами — да.

**Как попробовать.** Попросить у Вадима share-ссылку (или дождаться выката репо) →
человеку открыть в браузере; агенту — `curl -H "Accept: application/json" <share-url>`
и дальше по манифесту.

**Известные ограничения (на 24.07).** snapshot.id без заявленной схемы каноникализации —
идентификатор версии, не доказательство «двое читали одно» (автору предъявлено, в плане);
«Copy for agent» промпт — удобство человеку, агенту входить контрактом, не исполнением
чужого текста; графовые descriptions — данные, не инструкции (инъекции).

**Статус.** Живой деплой, протокол развивается быстро по фидбэку. Запланирован
двусторонний слепой тест переносимости (Арет ↔ Praxis: экспорт реальной развилки,
слепое восстановление без автора). Итоги — сюда.
